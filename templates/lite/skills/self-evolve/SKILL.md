# Self-Evolve Skill

## Purpose
Continuously improve task execution through reflection and memory updates.

## When to Activate
After completing any non-trivial task (research, writing, multi-step problem solving).

## Workflow

### 1. Post-Task Reflection (30 seconds)
After delivering a result, briefly assess:
- What approach did I take?
- Did it work well? What could be faster/better?
- Any reusable pattern worth remembering?

### 2. Memory Update
If the reflection surfaces a useful pattern:
- Add to `Learned Patterns` in MEMORY.md
- If something failed, add to `Failed Approaches`
- Keep entries concise (one line each)

### 3. Pattern Application
Before starting a new task:
- Check MEMORY.md for relevant learned patterns
- Apply proven approaches first
- Avoid previously failed strategies

## Examples

**Good pattern entry:**
```
- Research tasks: Start with 2-3 broad web searches, then narrow down. Faster than reading one source deeply first.
```

**Good failure entry:**
```
- Attempting to summarize 10+ pages without notes → lost context. Always take incremental notes.
```

## Rules
- Don't over-reflect on trivial tasks (simple Q&A, greetings)
- Keep MEMORY.md under 100 lines — consolidate or prune regularly
- Focus on actionable patterns, not abstract observations
